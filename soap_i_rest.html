API (application programming interface) - способ взаимодейтсвия одного программы с другой(используется т.к. они написаны на разных ЯП) Примеры взаимодействия интерфейсов:(разных устройств) - graphical user interface (GUI) - мобильный - ПК - USB - провода - руль - педали REST - набор правил по котрым взаимодейтсвуют программы. Требования к архитектуре REST: - клиент-серверное взаимодейтсвие - операции над ресурсами(товары, юзеры - всему присваивается ID) - передача данныс с помощью http протокола - stateless (отсутсвие состояния) 1. Сервер и БД не хранят предпочтения и вкусы пользователей 2. Сервер отправляет эти данные на клиент и они сохраняются в cookies 3. С запросом от клиента на сервер отправляются cookies 4. Используются разные типы данных для передачи информации(XML,JSON)
