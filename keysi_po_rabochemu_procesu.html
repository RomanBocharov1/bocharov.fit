Чарльз - сниффер, кроссплатформенное ПО прокси-сервера для отладки HTTP, написанное на Java.
Позволяет пользователю просматривать HTTP, HTTPS и активированный трафик TCP-порта
Сюда входят запросы и ответы, включая HTTP-заголовки и метаданные (cookie, кэширование и кодирование информации)
Сниффинг — процесс мониторинга и перехвата всех пакетов, проходящих через сеть, с помощью инструментов сниффинга.

-нет ли дублей запросов

-ломка верстки
-тест разной скорости передачи данных
-отключение кеша
-отключение куки
-переадресация запросов с одного URL на другой,
(можно скопировать юрл вставить в поля в чарльз и нажать таб, само заполнится)

Подмена данных - Rewrite
Создаем правила, которые изменяют запросы и ответы, когда те проходят через Charles Proxy.
Можно добавлять,изменять заголовок, искать и заменять текст в теле ответа или запроса, и т.д.
Пример: Хотим изменить количество бонусов юзера(для ломки верстки)
1.Tools → Rewrite → галочка «Enable Rewrite» → Add.
2.В поле Name - название подмены (Change bonus)
3.Location → Add , вводим хост и путь
4.В поле ниже add, создаем Rewrite Rule
Type: Body (параметр находится в теле);
Where: Response или request ( параметр находится в ответе от сервера);
Раздел Match: поле «Value» укажите значение и параметр, который возвращает сервер сейчас
Раздел Replace: поле «Value» укажите значение и параметр, который нужно вернуть
5.Отправить запрос
это автоматический способ менять данные

Throttle Settings - задаем разные параметры скорости соединения с выбранным доменом.
1.Proxy → Throttle Settings → галочка Enable Throttling.
2.«Only for selected hosts», задаем определенный хост, к которому будут применяться настройки.

Параметры на всякий случай:
Bandwidth — максимальный объем данных, который может быть передан с течением времени.
Utilisation — доля общей пропускной способности, которая может быть предоставлена пользователю в любой момент времени.
Latency — задержка в миллисекундах по запросу firts между клиентом и удаленным сервером.
MTU — максимальное передающее устройство для текущего пресета.
Reliability — мера вероятности, что соединение не удастся. Используется для имитации ненадежных сетевых условий.
Stability — мера вероятности, что соединение будет нестабильным и, следовательно, снизится качество.
Полезно для моделирования сетей, в которых периодических падает качество связи, например, мобильных.

Reverse proxy — обратный прокси-сервер.
Используется для того, чтобы принимать запросы из Интернета и перенаправлять их на один из веб-серверов.

Port Forwarding — проброс портов (перенаправление портов/туннелирование) — процесс пересылки трафика,
адресованного конкретному сетевому порту, с одного сетевого узла на другой.
Этот метод позволяет внешнему пользователю достичь порта внутри локальной сети.

MacOS Proxy/Windows Proxy -проксирование трафика с вашего веб-браузера.

No Caching(вкладка tools) - кэширование, манипулирует заголовками HTTP, которые управляют кэшированием ответов.

Block Cookies - заголовок файла Cookie удаляется из запросов, предотвращая отправку значений файла из клиентского приложения
(например веб-браузер) на удаленный сервер.
А также из ответов удаляется заголовок Set-Cookie, предотвращая получение клиентским приложением запросов на установку файлов cookie с удаленного сервера.
В настройках можно включить удаление Cookie как для всех хостов, так и для выбранных.

Map Remote — переадресация запросов с одного URL «Map From» на другой «Map To».
Подменяет хост, путь целиком или только параметры в зависимости от вашей задачи.
Например, с prod-сервера на dev-сервер.

Block List — блокировка доменных имен.
Когда веб-браузер попытается запросить любую страницу из заблокированного доменного имени, она заблокируется.
Можно выбрать либо «Drop connection», либо возврат 403 ошибки.

DNS Spoofing — перенаправляет доменное имя на определенный IP-адрес.
Виртуальный хостинг — это когда у вас есть несколько сайтов на одном IP-адресе, и веб-сервер определяет,
какой сайт вы запрашиваете, основываясь на имени, введённом в браузере.
Точнее, сервер смотрит на заголовок хоста, отправленный в запросе. Например, когда нужно подменить хосты, чтобы при
вводе какого-либо адреса в браузере (скажем, api.youla.ru) запросы уходили по другому адресу (допустим, на тестовую площадку).

Mirror — автоматически сохраняет все ответы, возвращаемые в Charles Proxy.
Они раскладываются локально в такой же иерархии, как на сервере. Если внезапно случился даунтайм на бэкенде,
отвалилась тестовая среда и т. д., у вас уже есть готовые моки для Map Local.
Tools → Mirror или Tools → Auto Save.

Compose — функция редактирования запросов, которые вы поймали.
1.Выбрать нужный запрос из списка, нажать на нём правой кнопкой и выбрать Compose.
2.Отредактировать
3.Нажать внизу «Execute», чтобы отправить запрос на сервер.

Recording Settings — настройки отображения списков разрешенных и запрещенных доменов.
В «Options» можно настроить лимит, то есть количество запросов, которое Charles Proxy может записать.
Во вкладке «Include» можно выбрать конкретный домен для отображения пакетов.
Во вкладке «Exclude» можем выбрать те домены, которые необходимо спрятать при сниффинге.

Focus(правая кнопка на запросе) — эта функция перемещает домен на первые позиции в списке.

Repeat(правая кнопка на запросе) — отправляет на сервер запрос, идентичный выбранному.

IP сервера в свегере в адресной строке

201 - Запрос успешно выполнен и в результате был создан ресурс.
204 - тело ответа пустое(данных нет), запрос выполнен успешно(успешная операция), но клиенту не нужно уходить со своей текущей страницы.
ошибка 401 - ошибка аутентификации( нет в бд данные доступа, которые ввел юзер)
ошибка 403 - ошибка авторизации(у юзера нет прав, чтобы просматривать этот ресурс)
ошибка 405 - не тот метод
ошибка 406 - сервер не может вернуть ответ.Заголовки,формат\кодировка страницы не те,проблема со стороны сайта обычно
ошибка 413 - файл, который вы пытаетесь загрузить на сервер слишком большой для заданной конфигурации сервера.
ошибка 415 - формат содержимого не поддерживается сервером.
если в теле ответа в постмане просто "1", значит еще не успели привязать или не закончили работу над данным запросом
